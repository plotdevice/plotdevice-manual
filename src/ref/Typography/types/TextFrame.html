TextFrame

Description

The Text objects you create using the text() command contain a series of one or more TextFrame objects which define the geometry used for layout. By default, each Text has only a single TextFrame with an "offset" position of (0,0) and the same width/height as the Text object itself.

You don't need to create TextFrames manually; instead you'll encounter them when working with Text objects. Calling a Text object's flow() method will create a linked series of frames and allow you to position and size each of them by looping over the values it returns. Once you've flowed a run of text, you can refine or inspect the geometry using the Text object's frames property.

Properties (read & write)

txt.x # horizontal offset from the Text object's origin
txt.y # vertical offset from the origin
txt.width  # maximum line width before word-wrapping
txt.height # maximum height for wrapped-lines

txt.offset # the frame's x/y as a Point relative to the parent Text's position
txt.size # the frame's width/height as a Size

Properties (read-only)

txt.idx # the frame's order within the flow sequence
txt.text # the portion of the Text's string displayed in the frame
txt.metrics # the Size of the text being displayed in the frame
