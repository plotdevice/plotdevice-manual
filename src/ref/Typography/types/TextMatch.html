TextMatch

Description

The Text objects you create using the text() command are still "live" objects that can provide information about their contents and layout. The Text object's content properties allow you to slice out portions of the text by returning TextMatch objects that describe their character and geometry data.

Each TextMatch object represents a contiguous range of characters in the Text object's string and describes their layout (whether it lies on a single line, a range of lines, or even across multiple TextFrame "columns". Depending on which property you retrieve the match from, different TextMatch properties will be relevant:

String Properties

match.text  # the substring that was matched
match.start # the index of the first character in the match
match.end   # the index of first character after the match

Drawing Properties

match.lines # a list of one or more LineFragments describing layout geometry
match.path  # a Bezier object with the glyphs from the matched range

Regular Expression Properties (via Text.find())

match.m # a regular expression Match object

XML Element Properties (via Text.select())

match.tag     # a string with the matched element's name
match.attrs   # a dictionary with the element's attributes (if any)
match.parents # a tuple with the parent, grandparent, etc. tag names
